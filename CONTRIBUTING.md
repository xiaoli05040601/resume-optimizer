# Contributing

Thanks for considering contributing to `resume-optimizer`.

## What Is Most Helpful

Contributions are especially valuable in these areas:

- better real-world job application examples
- clearer bilingual phrasing
- improved evaluation cases
- stronger resume-writing heuristics for different industries
- better host-environment compatibility notes

## Before You Submit Changes

Please keep these principles in mind:

- do not fabricate candidate experience
- keep advice concrete and actionable
- prefer structured outputs over vague prose
- preserve bilingual readability when editing public-facing docs

## Suggested Workflow

1. Fork the repository.
2. Create a focused branch.
3. Make your changes with a clear scope.
4. Update docs or evals if behavior changes.
5. Open a pull request with a concise explanation of what changed and why.

## If You Add New Examples

Please make sure the examples:

- do not contain real private personal data
- include realistic job or event constraints
- show why the output should be considered good

## If You Change The Skill Logic

Please update at least one of:

- `SKILL.md`
- `promotion.md`
- `evals/evals.json`
- `README.md`

That keeps the repository aligned across implementation, positioning, and evaluation.
